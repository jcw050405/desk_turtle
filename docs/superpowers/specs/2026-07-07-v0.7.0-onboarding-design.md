# v0.7.0 Onboarding Design

## Goal

Add a first-run readiness summary that tells users whether hardware, camera, and Supabase/social setup are ready, and replace raw sync status values with user-friendly labels.

## Scope

- Add a dashboard-style onboarding card at the top of the app content.
- Summarize hardware connection, camera check state, and social/Supabase profile/group state.
- Provide buttons that switch to the hardware, monitor, or social tab when setup is missing.
- Display `local_only`, `pending_sync`, and `synced` with friendly labels and short explanations.
- Do not read, print, or commit `.env`.
- Do not require real hardware, camera, installer, or Supabase checks in automation.

## Architecture

Create a small `onboardingStatus` service with pure functions for status labels, readiness rows, and overall readiness. `AppShell` owns the active tab and polls lightweight status sources through existing `serialClient` and `settingsClient`. `LocalHistory` uses the same sync label helper so the wording is consistent.

## Testing

Use Node tests with `tsx` to verify pure status logic before wiring it into React. Full verification remains the existing project commands: `test:node`, `test:mvp2`, `lint`, and `build`.
