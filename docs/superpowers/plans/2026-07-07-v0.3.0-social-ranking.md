# v0.3.0 Social Ranking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the existing social ranking flow usable by adding pending sync retry, current-user rank emphasis, and manual ranking refresh.

**Architecture:** Keep Supabase calls in `socialClient`, sync orchestration in `sessionSync`, and renderer UI state in the existing page components. Persist retry results through the existing Electron session API so local history updates without adding a new IPC surface.

**Tech Stack:** Electron, React, TypeScript, Supabase RPC, node:test.

## Global Constraints

- Do not print or commit `.env` values.
- Hardware, camera, installer, and real Supabase checks are user checklist items.
- Prefer `npm.cmd run test:mvp2`, `npm.cmd run lint`, and only build Electron if needed.
- Current firmware servo pin remains GPIO3.
