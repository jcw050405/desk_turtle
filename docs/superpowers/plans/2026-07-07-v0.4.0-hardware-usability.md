# v0.4.0 Hardware Usability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the ESP32-C3/MG90S hardware screen easier to understand and more resilient when the serial connection changes.

**Architecture:** Keep serial transport in Electron unchanged, add renderer-side presentation helpers for firmware responses, and poll serial status from the hardware settings page while it is mounted.

**Tech Stack:** Electron, React, TypeScript, node:test.

## Global Constraints

- Servo signal pin remains GPIO3.
- Firmware ACK values remain `ACK:BAD:80` and `ACK:GOOD:10`.
- Do not require real hardware verification in automation.
- Real hardware and USB-disconnect checks are user checklist items.
