# TurtleGuard v0.1.0 Release Checklist

Date: 2026-07-01
Status: release candidate validated locally

## Scope

This release is the first local Windows desktop MVP for TurtleGuard.

Included:
- Windows Electron installer generation
- Packaged renderer loading from `renderer-dist`
- Camera posture monitor flow
- Calibration-based GOOD/BAD posture judgment
- Local session draft/recovery storage
- Arduino serial port listing, auto-connect, manual connect, disconnect
- Servo test and posture-state serial output

Not included:
- Public social ranking service
- Supabase-backed friend/group features
- Auto update
- Custom app icon and code-signing certificate
- macOS/Linux installers

## Generated Installer

Expected installer path:

```text
.\turtleguard\dist\turtleguard Setup 0.1.0.exe
```

Expected app metadata:
- Package name: `turtleguard`
- Product name: `TurtleGuard`
- Version: `0.1.0`

## Automated Checks

Run before publishing or sharing the installer:

```powershell
cd .\turtleguard
npm run lint
npm run test:node
npm run build:electron
```

Expected result:
- TypeScript check exits with code 0
- Node tests pass
- Electron Builder creates `dist\turtleguard Setup 0.1.0.exe`

## Manual QA Checklist

Ask the user to run this checklist instead of spending model tokens on visual/hardware validation.

Installation:
- [ ] Uninstall old `react-example` build if it is still installed.
- [ ] Install `turtleguard Setup 0.1.0.exe`.
- [ ] Confirm the installed app launches without a white screen.
- [ ] Confirm the app title/product is shown as TurtleGuard or turtleguard, not `react-example`.

Camera and posture:
- [ ] Grant camera permission when prompted.
- [ ] Confirm the camera preview appears.
- [ ] Run calibration in a normal upright posture.
- [ ] Confirm upright posture is judged GOOD.
- [ ] Move into turtle-neck posture and confirm BAD appears.
- [ ] Return upright and confirm GOOD appears again.

Hardware:
- [ ] Connect the Arduino USB cable.
- [ ] Open hardware/settings screen.
- [ ] Click refresh and confirm a COM port candidate appears.
- [ ] Click auto-connect and confirm connected status.
- [ ] Run servo test and confirm the turtle hardware moves.
- [ ] Confirm BAD posture sends the hardware output signal.
- [ ] Confirm GOOD posture sends the recovery signal.

Session reliability:
- [ ] Start a posture session.
- [ ] Keep the app running for at least 3 minutes.
- [ ] End the session and confirm local history records the result.
- [ ] Force-close and reopen the app during a session, then confirm recovery does not crash.
- [ ] Close the app and reopen it, then confirm the COM port is not locked.

Result notes:
- [x] User reported the installed exe launches and works after packaging fixes.
- [x] User reported manual inspection was completed before this release checkpoint.

## Known Release Risks

- Windows may warn users because the app has no paid code-signing certificate yet.
- The app currently uses the default Electron icon.
- The installer is Windows-only.
- Social features are intentionally deferred until the local hardware/posture MVP is stable.
- Some Arduino clones may appear as CH340/USB-SERIAL rather than Arduino by name; auto-detect should still prefer likely ports.
